Projects

All projects are initial idea blurbs - they are negotiable and should be discussed.

1. A Filter Bubble for Trusted News: Semi-Automated Social Fact-Checking

Type: software engineering

In this project you will build a browser plug-in which lets users label and annotate news articles with evidence of their veracity, or lack thereof. The system will also let users register accounts and assess other users as trustworthy or untrustworthy. This labelling will allow the system to propagate trustworthiness metrics along the graph of users, based on some existing graph algorithms in abstract argumentation theory.

2. A Searching, Storage and Annotation Engine for Researchers

Type: software engineering

You will use the CORE API in order to build a search engine client that lets the user annotate articles and create a personal database. The search engine client will let the user add a paper to their personal collection, which will download them into an offline document collection where the paper will be indexed, along with their annotation, for further use.

3. An Interpretable Active-Learning Based Dialogue Labeling Application

Type: research and software engineering

Creating a good dataset is one of the major pain points when investigating new venues in machine learning. One of the reasons for this is that labelling data is difficult and time-consuming, and software designed to ease that process is scarce and expensive. This project aims at producing an open source alternative to those solutions, where the designer of the experiment can configure a hosted personalised labelling software in a couple of hours. The application would then allow authorised users to log in a Web interface, and be presented with a sequence of random examples that they can label, which is where the research part of the project is. The goal of the project is to apply active learning and interpretable classification in order to produce an online software that allows people to label turn-by-turn dialogues with a generic set of classes.

4. A Cross-Lingual Approach to Fake News Classification

Type: research

In this project, you will investigate fake news classification in a new angle by asking the following fundamental question: can we use data in multiple languages to enhance the performance of a fake news classifier?

5. A news aggregator and analyser to follow the evolution of public sentiment through time and space

Type: software engineering

Note: This project will be supervised by me, and co-supervised by Elena Nichele

In this project you will build an application that will track, download and aggregate news about a specific set of topics in a pre-specified set of British newspapers. This application will provide a dashboard that allows the user to track the subtopics being discussed as well as the overall sentiment expressed in the news articles based on time (publication date) and space (allowing the user to specify a location for each news outlet). It will be evaluated by real linguists and scientists and might end up being part of a larger project, and a potential scientific publication, to which you will have the option to take part in (provided the project is successful).

6. A dashboard for collocation analysis in large datasets

Type: software engineering

Note: This project will be supervised by Elena Nichele and co-supervised by me

Non-native speakers of English are very likely to struggle with word order and semantic shades, i.e. selecting the most suitable word for a given context. This problem affects speakers at all levels of proficiency and is particularly important to achieve a native-like command of the language. In this project you will build a messaging application to enhance language learning by tracking collocates over time. The application should allow a language learner to communicate with a native speaker and compare the use of collocates over time in order to see whether the learner is improving (the learner progressively models the native and their use becomes increasingly similar to that of the native speaker), stable (their use does not change) or getting worse (the learner does not absorb from the native and their use remains distinguishable from the one from the native speaker). The application should then display regular feedback in the form of a traffic light system (green/orange/red, or some colourblind-accessible variation) so that the learner knows whether they are going in the right direction, and a plot to visualise their progress over time. Finally, it should point out specific examples that determined the outcome, so that the non-native speaker could learn from their own mistakes or inaccuracies.
